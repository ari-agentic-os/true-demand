## 🔴 硬伤

### 0. 不会读用户已有的文档 / MVP 原型
**现状：** `/开局` 是「从零问创始人」。对一个已经有 PRD、设计文档、跑起来的原型的人，会一条条重复盘问他文档里已写的东西。
**为什么重要：** 这正是 skill 最该有的杀手锏——人做不到、AI 能做的，是从一份功能堆满的 PRD 里**揪出那条没人写、却最致命的付费假设**。
**修法（Step 0 — 先吃上下文）：** 扫描本地 PRD / README / specs / 设计文档 + 原型的页面与功能清单 → 抽取隐含假设（目标用户、核心价值、**付费假设**、他自认为已验证的部分）→ 草拟 `看板_核心假设.md` → **只问文档没答的缺口**。
**要守住的哲学：** 读文档读到的是「说（say）」，不是「做（bill）」。扫描产出的是**待验证假设**，不是验证结论；并要反问「你说已验证的那条，是 desirability 还是 viability 的证据？」

### 1. 招人比问问题难得多，skill 几乎不管
**现状：** 给一份访谈提纲然后说「去找 5 个真实用户聊」。
**为什么重要：** 创始人卡死在**怎么约到对的人**——去哪找、怎么写邀约、给不给报酬、怎么防鸽。提纲是 10% 的活，招募是 90%。
**修法：** 加招募工具包——按 persona 给「去哪找」清单、可直接发的邀约模板（冷启 DM / 邮件 / 社群帖）、激励与排期建议、防放鸽子话术。

### 2. 本质靠访谈，低配了行为实验
**现状：** 主推 Mom Test 访谈；定价靠 Van Westendorp PSM。
**为什么重要（最伤）：** 核心公理是「读账单不读客套（say ≠ do）」，但访谈拿到的仍是**优化过的「say」**。最硬的 viability 证据是**真实掏钱动作**——预售、fake-door、Stripe 预收、落地页烟雾测试、定价页点击率。而 **Van Westendorp 本身就是「问出来的报价」（stated preference）**，恰恰是它要反对的。
**修法：** 把「行为实验」升为一等公民。`/开局` 在出访谈提纲的同时，给一条**行为实验**建议（按阶段：预售 / 烟雾测试 / 假门 / 并发定价），并明确「真实付费 > 访谈口头意愿」的证据权重。

### 3. 分析的是创始人自己记的笔记 → 垃圾进垃圾出
**现状：** `/纠偏` `/确诊` 分析创始人粘进来的二手笔记。
**为什么重要：** 创始人有确认偏误——选择性记录、诱导用户、把「支持你」记成「愿意付费」。skill 看不到真实访谈。
**修法：** 支持录音/逐字稿粒度的输入；加「反审创始人」环节（你到底有没有按行为问法问？有没有诱导？）；给笔记模板，现场只记行为与原话，不记结论。

## 🟠 方法论软肋

### 4. 「5 个人就够」把可用性定律错用到需求验证
Nielsen 五人定律是给 **usability testing** 的，不是 demand / WTP。B2C 与 B2B 量级不同，单 segment 5 样本可能分不清真信号与噪音。
**修法：** 样本量按目标（可用性 vs 需求 vs 定价）和 B2B/B2C 分档；「5 个就停」改为「按 segment 达到信号饱和才停」。

### 5. 做/转/弃判定线太模糊
5 个里要几个有真实付费行为才算「做」？阈值不清。ABOUT.md 自己承认要「跑 5 次看一致性」，等于默认了不稳定。
**修法：** 给可证伪的决策规则（如：≥N/5 出现过去 12 个月真实相关支出，且能复述具体金额/场景 → 做；否则转/弃），并要求引用原话作证据。

## 🟡 覆盖盲区

### 6. 两头接不住
scope 卡死在「成长期：有种子用户但卡住」。**pre-MVP 完全没用户**（str-validate 原始人群）和 **B2B/企业级 WTP**（采购、用人 vs 掏钱人、ROI、报销文化）拿到的方法不合身。
**修法：** kickoff 增加一道「形态判断」分支，pre-MVP 走需求/烟雾测试路线，B2B 走 champion + 预算主 + ROI 路线。

### 7. 没有迭代闭环 + 不吃已有量化数据
验证是循环（聊→学→改假设→再聊），但「确诊之后下一个实验是什么 / pivot 之后要不要从零再验」不是一等公民。且成长期创始人手里常已有客服工单、留存曲线、竞品差评、社群信号——skill 一律不吃。
**修法：** `/确诊` 输出「下一轮实验」并支持回灌到看板形成多轮；吃已有量化数据做证据三角验证（与 #0 同源）。

## 🟢 元风险

### 8. 交棒后沉默 = skill 自己的活跃度问题（反讽）
skill 在 `/开局` 后退场，把最难、最让人害怕的「去跟陌生人聊」丢给用户，中间两周无陪伴/督促/模板——大概率用户一去不回。也就是说，这个 skill 很可能**过不了它自己出的 viability 测试**。
**修法：** 更轻的第一步（先约 1 个人而不是 5 个）、邀约/记录模板降低启动门槛、可选的进度回访提醒。
