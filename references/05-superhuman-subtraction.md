# 05 · Superhuman Core-Path Subtraction (rescue low activation)

When the wound is **low activation/retention** — people download, poke, and drift — the reflex is to add
features. Usually wrong. The fix is **subtraction**: find the one core path your *most engaged* users love,
and cut almost everything else so a new user hits that value faster. 弱水三千，只取一瓢。

> Sources: Rahul Vohra (Superhuman) — the PMF engine built on the Sean Ellis survey; Sean Ellis, "How would
> you feel if you could no longer use this product?"; Des Traynor / Intercom on product surface reduction.

## 1. Run the Sean Ellis PMF survey on real users

Ask everyone who has used it recently: **"如果以后用不了这个产品，你会有多失望？/ How would you feel if you
could no longer use this product?"** — 很失望 / somewhat / 不失望 (very / somewhat / not disappointed).

- **≥ 40% "very disappointed"** = you have a PMF core to amplify. Below that, you're looking for it.
- This metric is only meaningful with real usage — which a growth-stage product has.

## 2. Segment on the "very disappointed" — your superusers

Ignore the indifferent for now. Look only at the people who'd be very disappointed:
- **Who are they?** (role, context) — this is your real target segment, often narrower than you hoped.
- **What's the one thing they love?** ("没了它我没法 ___" / "I couldn't ___ without it") — the **core path**.
- **What benefit do they get that others miss?** — the activation moment to engineer everyone toward.

## 3. Subtract toward the core path

- **Cut or bury** everything the superusers don't name. Each extra feature is a tax on the activation moment.
- **Rebuild onboarding** so a brand-new user reaches the superusers' core value on the first session — not after exploring a menu.
- **Then, only then, add** — but only what moves the "somewhat → very disappointed" group up. Roadmap = close
  the gap to the core value, not pile new surface on.

## 4. Output (feeds `02-diagnose.md`)

- The core path (one sentence) + the superuser segment.
- A cut-list (features to remove/hide) and the single activation moment to engineer toward.
- The current "very disappointed" % as a baseline to move.

> Rule of thumb: if activation is the problem, your next release should make the product *smaller and faster
> to value*, not bigger.
