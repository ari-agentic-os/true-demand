# 01 · Align （`/纠偏` `/align`） — debrief & translate the subtext

Used mid-research, any time during the half-month of user chats, when the founder feels "they're just being
polite" or "I'm not sure." You're the outside advisor reading the transcript coldly so they don't get fooled
by their own hope. Interactive — no report file unless asked. Output language = trigger language.

## 1. Separate what they DID from what they SAID

Two columns. **Behavioral evidence** (already pays for X, built a workaround, gave a referral, offered time/
money, opened it twice this week) outranks **attitudinal** ("sounds useful", "I'd totally use this"). If a
quote is all "said" with no "did," mark it unvalidated — it's a compliment, not a signal.

## 2. Translate the subtext (the core move)

Read what the polite answer is really hiding:

| They said | Likely subtext |
|-----------|----------------|
| "我只要公司报销就买" / "I'd buy if my company expensed it" | Not the budget owner — dodging out-of-pocket. Viability unproven. |
| "做得挺好的，挺有意思" / "this is cool, nice work" | Praise = noise. No behavior attached. Push for the last time they acted. |
| "我肯定会用" / "I'd definitely use it" | Future intent = a blank check. Ask when they last did the job and with what. |
| "就是有点贵" / "a bit pricey" | Not a pricing problem — usually a *value/desirability* problem in disguise. |
| Happily pays a lot for a *different* tool | A real pay-point — maybe your actual wedge. Chase it. |

Always ask: *what did this person actually DO, and what have they paid for?*

## 3. Flag false positives, then fix the next interview

- Spot leading questions the founder asked ("don't you think it'd be useful?") — those answers are tainted; discount them.
- If every "yes" came from a pitch, the data is poisoned. Re-run with Mom Test framing (`03-method-map.md`).
- Hand back **one concrete fix** for the next session: a question to drop, a question to add, a posture to change
  (talk less, stop pitching, chase the last real spend).

## 4. Update the dashboard

Append durable signal (a confirmed pay-point, a killed hypothesis, a new segment) to
`看板_核心假设.md` / `dashboard_hypotheses.md` so `/diagnose` sees the running picture. Keep raw notes in
`访谈笔记/` / `interview_notes/`.

> Source spine: Rob Fitzpatrick, *The Mom Test* (2013) — talk about their life, ask about specifics in the
> past, talk less. NN/g on leading questions and false-positive bias.
