# 💊 付费意愿清醒剂 · true-demand  <sub>成长型创始人版</sub>

> 专治「内测个个好评，一收费全员卸载」的创始人自嗨症。
> 它从用户的**账单**里读真话，不听他们嘴上的客套。

一个面向 [Claude Code](https://claude.com/claude-code) / Agentic OS 的技能，给**已经有种子用户、但卡住了**的成长型创始人用：活跃度低、一谈收费就卡壳、不知道下一步该加功能还是换方向。它沉稳、不说大厂黑话、不顺着你说好话，把你的注意力从「功能好不好用」拉回到那条最该测、几乎没人测的红线——**商业可行性（用户究竟愿不愿意掏钱）**。

## 它解决什么问题

成长期最危险的不是没人用，是**虚假繁荣**：小红书求来的内测、Product Hunt 的 upvote，都是成本极低的客套。下载量很高、活跃度极低，往往说明你掉进了「工具类自嗨」——而真正的死因在收费那一刻才暴露。

核心信条：**嘴上说的 ≠ 实际做的，而大多数人只测了「嘴上说的」。**

## 三大核心指令（中英镜像，文件持久化）

抛弃「单窗口死磕」，拆成三个随时单点触发、基于本地文件的硬核动作。你敲哪种语言，它就用哪种语言在本地落盘：

| 中文 | 英文镜像 | 干什么 | 产出 |
|---|---|---|---|
| **`/开局`** | **`/kickoff`** | 诊断现状、卡死最大商业风险、下发防客套访谈提纲，然后退场让你去聊 | 看板 + 访谈提纲 |
| **`/纠偏`** | **`/align`** | 半个月调研途中，丢进笔记，帮你刺穿用户潜台词、防假阳性 | 聊天窗交互 |
| **`/确诊`** | **`/diagnose`** | 聊完 5 个用户，跨文件综合所有笔记，给你做/转/弃终局报告 | 终局确诊报告 |

完整运作流程见 [`docs/运作流程.zh.md`](./docs/运作流程.zh.md)。

## 八大底层方法论

口语是大白话，底层严格执行行业黄金标准：

| 维度 | 方法论 | 在工作流里的卡点动作 |
|---|---|---|
| 商业红线 | IDEO 三透镜 | `/开局` 卡死 80% 权重在商业可行性，砍掉可用性自嗨 |
| 防客套测谎 | The Mom Test | 过滤未来意愿客套，逼出账单里的过去真实行为 |
| 动机挖掘 | JTBD | 拆出购买背后的社会/情感痛点与切换阻力 |
| 定价破局 | Van Westendorp PSM | 推导不流失内测粉的定价黄金带 |
| 信号收敛 | 尼尔森五人定律 | 锁死 5 个深访达 85% 信号饱和，终结无休止调研 |
| 功能迷茫 | Teresa Torres 机会解决方案树 | 把杂乱笔记规整为 目标-机会-方案-实验 |
| 挽救活跃 | Superhuman 核心路径减法 | 靠超级用户反馈做减法，拯救低活跃 |
| 代码救赎 | 最小技术资产留存转向 | 证伪后不浪费一行老代码，转向高客单价场景 |

设计原理与「怎么验证这个技能本身」见 [`ABOUT.md`](./ABOUT.md)。


## 安装

把本仓库内容放进技能目录 `.claude/skills/true-demand/`，技能会按需读取 `references/`。
中文产出经 `humanizer-zh` 去 AI 味，英文产出经 `tool-humanizer`。可选增强：`mkt-icp`、`mkt-positioning`。
