# 💊 付费意愿清醒剂 · true-demand

> **成长型创始人版** · 专为 Claude Code 打造的真实付费意愿验证技能。

你的 MVP 已经拿到了第一批种子用户，但很快遇到了瓶颈：用户活跃度低？一谈收费就卡壳？不知道下一步该加功能还是换方向？

本技能（内部名 `true-demand`）专为解决「成长期的自嗨式调研」而生，帮你用科学的市场行为，测出种子用户账单里的真话。它会把你的注意力从「功能好不好用（可用性）」，稳稳拉回到最常被逃避的那条线——**商业可行性：用户究竟愿不愿意为此掏钱**。

## 🎬 1 秒体感：端到端使用案例 (End-to-End Example)

> 💡 许多初创项目死于「自嗨」，看看这个 Skill 是如何帮你快速「退烧」的：

```text
用户：
/开局 我做了一个面向独立创作者的"多平台排版优化与一键同步"的浏览器插件。在小红书和即刻发了内测，有 500 多个写小红书、公众号的博主留邮箱拿到了内测包。但现在发现大家下载后，大部分人只用它来"本地排版美化"，很少有人真正去点那个"一键同步到多平台"的核心功能（活跃度很低）。下个月我打算上线收费，每个月 15 块钱，不知道会不会直接死掉，怎么测？

AI（true-demand）：
拿到 500 个创作者的邮箱是一个经典的"操作成本错觉"。创作者们对"能让排版变好看"的工具有天然的尝鲜欲，留邮箱只是她们在对"写出爆款"做低成本的工具投资。现在她们"只排版、不同步"，说明你可能把一个低频的刚需（跨平台分发），做成了高频的附庸（本地美化工具）。

而且你其实已经握着最硬的证据：500 个人里大部分只本地排版、几乎没人点"一键同步"——这是真实行为数据（不是嘴上说的），它在喊：你可能把一个低频刚需（跨平台分发）做成了高频附庸（本地美化）。

别盲目开订阅。我在本地建了三个文件：
① 验证看板《看板_核心假设.md》——最大假设「创作者愿意每月掏 15 元省掉手动跨平台搬运的痛」，并预设了确认线 / 放弃线；
② 防客套访谈提纲《提纲_真需求访谈.md》——含一封"向核心作者请教工作流"的破冰邀约 + 5 个看穿客套话的行为追问；
③ 行动计划《行动计划_招募与实验.md》——怎么约到对的人，外加一个并行跑的行为实验：给"一键同步"挂个 fake-door / 预售（点了就弹"¥15/月解锁"），用真实点击和付款来测，而不是嘴上问。

这里有一个经典的"辨真话"提问示范：

❌ 不要问： "如果之后同步功能稳定了，一个月收 15 块钱，你愿意订阅支持一下吗？"（创作者会出于同行鼓励说"不贵、支持"，但付钱时立刻假装看不见）。

✅ 应该问： "上个月为了把内容发到不同平台，或者为了排版好看，你真正花过最贵的一笔钱或者时间成本是什么？（比如买过什么排版主页的 VIP、用过什么付费分发软件、或者花钱找过兼职助理）当时为什么愿意花这个钱？现在的工具帮你省掉这个成本了吗？"

⏸️ 第一步别贪多：从那 15 个"每周打开、却从不点同步"的人里**先约 1 个聊**（约成了再扩），同时把 fake-door 挂上去。给自己定个执行意图——比如"周三前发出 3 封邀约"。哪怕只聊到 1 个、只丢 1 条笔记，就能用 `/纠偏` 一起看她到底卡在哪。
```

## 🛠️ 安装与快速开始

本技能是纯 Markdown（`SKILL.md` + `references/`），**同时兼容 [Claude Code](https://claude.com/claude-code) 与 Codex**，无需安装任何依赖即可运行。

### 方式一：Git Clone（推荐）

```bash
git clone https://github.com/ari-agentic-os/true-demand.git
```

把整个文件夹重命名为 `true-demand`，放进对应工具的 skills 目录：

| 工具 | 放置路径 |
|---|---|
| Claude Code | `~/.claude/skills/true-demand` |
| Codex（macOS / Linux） | `$CODEX_HOME/skills/true-demand`（默认 `~/.codex/skills/true-demand`） |
| Codex（Windows） | `C:\Users\<用户名>\.codex\skills\true-demand` |

放好后**重启一次会话**，技能即可被识别。

### 方式二：ZIP 下载

在 GitHub 点 `Code → Download ZIP`，解压后把文件夹重命名为 `true-demand`，放进上表对应目录，重启会话。

### 方式三：直接喂 Prompt（任意 AI 工具，零安装）

不想装也行：把 `SKILL.md` 全文复制进对话，开头加一句——「请扮演一个面向成长型创始人的沉稳产品顾问，严格按以下规范工作」，再粘贴 `SKILL.md` 内容即可。

### 用起来

- **三个入口：** 中文敲 `/开局`、`/纠偏`、`/确诊`；英文敲 `/kickoff`、`/align`、`/diagnose`。你敲哪种语言，它就用哪种语言在本地落盘（绝不中英夹杂）。
  - **Claude Code：** 直接当原生指令用，或描述处境自动命中。
  - **Codex：** 用 `/skills` 菜单选 `true-demand`、用 `$true-demand` 提及、或直接描述你的处境（命中 description 自动隐式触发）；`/开局` 等动词照打也能命中。若没给出明确中/英动词，技能会先问一次用哪种语言。
- **去 AI 味（可选依赖）：** 若另外装了 `humanizer-zh`（中文）/ `tool-humanizer`（英文），产出落盘前会自动过一遍去 AI 味；没装也能正常用（技能会自己手工润色）。

## ⚡ 三大核心指令（中英镜像，文件持久化）

为了顺应成长期长达半个月甚至一个月的真实调研周期，它抛弃了「在单一 Chat 窗口里硬耗」的模式，拆解为三个随时可单点触发、基于本地文件持久化的动作：

### 🚀 `/开局` · `/kickoff` —— 诊断现状与下发工具
- **什么时候用：** MVP 积攒了一批内测用户，但面临活跃度低、不愿付费或功能排期迷茫时。
- **它在干嘛：** ⓪ **先读你已有的东西**——repo 里有 PRD / 文档 / 原型**和已有数据（留存/工单/支付）**就先扫一遍，从文档抽隐含假设（真实数据则是最硬证据），只问没答的缺口，并判断你的形态（pre-MVP/成长期 · B2C/B2B）来调整方法；① **看穿表象风险**——把注意力从「可用性」稳稳拉回最常被逃避的「商业可行性」；② **本地落盘**——验证看板 `看板_核心假设.md` + 防客套访谈提纲 `提纲_真需求访谈.md` + **行动计划**（招募清单 + 一个行为实验）`行动计划_招募与实验.md`。
- **⏸️ 默认停点：** AI 在这里退场——**第一步要极小：先聊 1 个人**，同时并行跑行为实验（真实掏钱 > 口头意愿）；1 条笔记就能回来跑 `/纠偏`。确诊后会开下一轮，直到你有底气开发或果断止损。

### 🎯 `/纠偏` · `/align` —— 访谈复盘与心智翻译
- **什么时候用：** 调研途中，觉得「用户好像在说客套话」或「心里没底」时。
- **它在干嘛：** 当你的场外顾问。你把原始笔记/录音丢进来，它帮你客观分析潜台词——比如「只要公司报销我就买」，本质上是在逃避自费，预算主体根本不是他；或者帮你发现用户愿意花大价钱买别的东西，从而撬开新的付费点；并实时修正你下一场访谈的提问姿势，防止你被假阳性信号欺骗。

### 🏁 `/确诊` · `/diagnose` —— 跨文件综合与战略转向
- **什么时候用：** 聊完 5 个真实用户，手里攒了一堆杂乱笔记时。
- **它在干嘛：** 跨文件读取 `访谈笔记/` 下所有记录，用亲和图聚类真实付费信号与高价值痛点，结合机会-解决方案树（Torres）给一份《报告_终局确诊.md》：**做**（付费路径跑通，下发排期）·**转**（现有方向是伪需求，但发现底层刚需，教你复用技术资产优雅转向）·**弃**（彻底证伪，及时止损）。

完整运作流程见 [`docs/运作流程.zh.md`](./docs/运作流程.zh.md)。

## 🧠 八大底层方法论

底层严格执行行业黄金验证标准：

| 验证维度 | 引入的方法论 | 在工作流中的卡点动作 |
|---|---|---|
| 商&#8288;业&#8288;红&#8288;线&#8288;卡&#8288;位 | IDEO 三透镜模型 | `/开局` 把 80% 权重锁定在商业可行性，先放下可用性 |
| 防&#8288;客&#8288;套 | The Mom Test | 过滤掉所有未来意愿的客套话，还原用户账单里的过去真实行为 |
| 底&#8288;层&#8288;动&#8288;机&#8288;挖&#8288;掘 | JTBD 岗位任务理论 | 拆解购买背后的社会与情感痛点，诊断切换阻力 |
| 定&#8288;价&#8288;策&#8288;略&#8288;破&#8288;局 | Van Westendorp PSM | 通过定性行为对比，推导出不流失内测粉的定价黄金带 |
| 信&#8288;号&#8288;收&#8288;敛&#8288;控&#8288;制 | 尼尔森可用性五人定律 | 锁死 5 个真实深度定性访谈，达 85% 信号饱和，终结无休止调研 |
| 解&#8288;构&#8288;功&#8288;能&#8288;迷&#8288;茫 | Teresa Torres 机会解决方案树 | 把杂乱笔记规整为「目标-机会-方案-实验」，理清功能排期 |
| 挽&#8288;救&#8288;活&#8288;跃&#8288;留&#8288;存 | Superhuman 核心路径减法 | 弱水三千只取一瓢，通过超级用户反馈做功能减法，拯救低活跃度 |
| 技&#8288;术&#8288;资&#8288;产&#8288;复&#8288;用 | 最小技术资产留存转向（Pivot） | 证伪后不浪费一行老代码，复用现有技术资产匹配新的高客单价场景 |

设计原理与「怎么验证这个技能本身」见 [`ABOUT.md`](./ABOUT.md)。
